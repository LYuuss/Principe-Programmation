# 1 - Service Web  

Deux approches différentes, toutes deux définissant une manière de developper des interfaces d'applications qui permettent les échanges de données en ligne entre applications web implémenté par différents langage de programmation. C'est l'interopérabilité.  
  
  - SOAP (Simple Object Access Protocol), a vu le jour en premier, est un protocole standard initialement conçu pour que des applications développées avec différents langages sur différentes plateformes puissent communiquer. Elle présente ainsi plus de contrainte.  
  - REST (Representational State Transfer), plus souple en laissant plus de liberté au developpeur.
